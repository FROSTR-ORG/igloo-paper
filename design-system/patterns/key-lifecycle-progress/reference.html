Protocol Boundary Note
Generation Progress is a host-side lifecycle progress pattern, not a peer-protocol trace. Rotate Keyset uses this pattern to show trusted-dealer share refresh work for the same group public key. Create Keyset uses this pattern to show new-key generation and initial share preparation.

2 of 4 phases
Implementation Mapping
Process Source Shares = decrypt sources, recover current profiles, verify same group config and same group public key.
Recover Existing Key = reconstruct the current signing key from threshold source shares.
Generate Fresh Shares = re-split the same signing key into fresh shares.
Prepare Onboarding Packages = local replacement materialization, local backup publish, remote payload prep, remote backup publish, and bfonboard emission.

Implementation Mapping
Generate New Key = create a brand-new signing key and group package.
Split Initial Shares = issue the initial threshold share set for the new keyset.
Prepare Shares for Setup = prepare the generated draft that Setup Profile will consume. This does not imply that the local share is already chosen or that remote bfonboard packages have already been emitted.
