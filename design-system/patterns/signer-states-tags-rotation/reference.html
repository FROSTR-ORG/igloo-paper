ROTATE KEYSET STATES
Keyset rotation in progress
Collecting outside-runtime sources and generating fresh shares — do not close this tab
Local replacement activated — distribution pending
Your device has a fresh local share, new share public key, and new profile_id. Distribute remaining bfonboard packages to remote devices.
Keyset rotation complete — distribution handoff finished
All remote bfonboard packages have been accounted for. Remote devices can now adopt through the standard onboarding flow.
Keyset rotation failed — no changes made
Generation could not complete. Your current configuration remains active.
Same group public key preserved
Keyset rotation preserves the same group public key. Share public keys and profile_id change. Host-local settings such as relays and permissions are not rotation invariants.
Keyset Rotation Terminology
Rotate Keyset = fresh device shares for the same group public key.
Existing Device = remote output label for an already-known member receiving a new onboarding package.
Keyset Replacement = new group public key. Not shown in this flow.
Input = source packages (bfprofile or bfshare, or nsec where supported). Output = bfonboard packages.
Share public keys and profile_id change. Group public key does not. Advanced: group_id may change with threshold or member changes.
Protocol Boundary
Share generation is an outside-runtime trusted-dealer computation from nsec or threshold source packages.
There is no dedicated runtime share-generation relay method.
Relay protocol resumes only when bfonboard packages are adopted through the standard onboarding flow.
